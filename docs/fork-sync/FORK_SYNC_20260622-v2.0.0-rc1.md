## Bug 与教训

> 同步过程中发现的问题、修复措施、经验教训。

### 渠道配置 API 迁移

- **决策**: 将 `env-settings` 扩展的渠道配置从自定义 `/api/env-settings/*` 后端迁移到上游官方 `/api/channels/*` 接口
- **方式**: 引入 `adapters/channel-adapter.ts`（Adapter Pattern），在扩展层做字段映射，不修改任何官方源码（零侵入）
- **文件**: `channels.ts` 标记为 `@deprecated`（凭据字段改为从上游 `credential_fields` 动态获取）
- **配置**: 需要在 `config.yaml` 中启用 `channel_connections.enabled: true`（Gateway 需重启生效）
